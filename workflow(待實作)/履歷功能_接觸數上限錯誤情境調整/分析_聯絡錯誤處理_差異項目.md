# 聯絡錯誤處理 — AI 智慧查詢 vs 舊版查詢履歷 差異項目

> 僅列出兩版 detail 處理結果**不同**的 error code。

## 共同測試資料

```js
receivers = [
  { idNo: '1001', snapshotId: 'snap-A', name: '王小明' },
  { idNo: '1002', snapshotId: 'snap-B', name: '李小華' }
]
```

---

## `00101` — 超過每日接觸上限

假設 `detail = { viewLimit: '100', viewNow: '98', detail: '已超過每日上限' }`，`candidates = [{ idNo: '1001' }, { idNo: '1002' }]`

### AI 智慧查詢 (`toOverViewErrorField`)

```js
const { detail: { viewLimit } } = error  // 只取 viewLimit

content: ["每日可接觸上限履歷 100 筆, 您已接觸 100 筆"]
```

### 舊版查詢履歷 (`getOverViewErrorMessages`)

**情境 A：尚有餘額**（viewRemain = 100 - 98 = 2 > 0）

```js
content: [
  "今日尚可瀏覽 2 位求職者，",
  "請取消勾選 0 筆履歷，以符合每日履歷查詢上限。",
  "王小明、李小華",
  "以上送出失敗"
]
```

**情境 B：無餘額**（viewNow = '100'，viewRemain = 0）

```js
content: [
  "已超過每日上限",
  "王小明、李小華",
  "以上送出失敗"
]
```

### 比對結果

| 項目 | AI 智慧查詢 | 舊版查詢履歷 (尚有餘額) | 舊版查詢履歷 (無餘額) | 是否相同 |
|------|-----------|---------------------|---------------------|---------|
| content | `["每日可接觸上限履歷 100 筆, 您已接觸 100 筆"]` | `["今日尚可瀏覽 2 位求職者，", "請取消勾選 0 筆履歷，以符合每日履歷查詢上限。", "王小明、李小華", "以上送出失敗"]` | `["已超過每日上限", "王小明、李小華", "以上送出失敗"]` | **不同** |
| 是否列出失敗者姓名 | 否 | 是 | 是 | **不同** |
| 是否區分餘額情境 | 否（固定文案） | 是（兩種分支） | 是（兩種分支） | **不同** |
| 已接觸數顯示 | 誤用 `viewLimit`（與上限值相同） | 正確使用 `viewNow` 計算 | 正確使用 `viewNow` 計算 | **不同** |

---

## `10000` — 自訂批次錯誤碼

假設 `detail = [{ idNo: '1001' }]`，`candidates = [{ idNo: '1001' }]`，`message = "部分聯絡失敗"`

### AI 智慧查詢 (`toOtherErrorField` — 走 default)

```js
content: ["部分聯絡失敗"]
```

### 舊版查詢履歷 (`getErrorModalProps`)

```js
content: ["王小明", "以上送出失敗"]
```

### 比對結果

| 項目 | AI 智慧查詢 | 舊版查詢履歷 | 是否相同 |
|------|-----------|------------|---------|
| content | `["部分聯絡失敗"]` | `["王小明", "以上送出失敗"]` | **不同** |
| 是否列出失敗者姓名 | 否 | 是 | **不同** |
| 是否有獨立 adapter | 否（走 default） | 是（命中 `BC_COMM_CUSTOM_ERROR_CODE`） | **不同** |
