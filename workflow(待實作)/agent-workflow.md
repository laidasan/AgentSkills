# Agent 工作流程

Note: 某次專案任務中，自己和 Agent 互動時，發現自己偏好的工作流程。

## Phase 1：討論

- 使用者提供：背景文件 + 需求 + 實作風格參考（具體檔案），一次給足脈絡
- Agent 回應：分析摘要 + 主動提出需要確認的問題、風險、建議
- 使用者回應：逐一確認方向性決策
- 結束條件：雙方確認沒有其他問題，使用者明確指示進入下一階段

## Phase 2：規劃

- Agent 產出 plan md 文件，包含兩個區塊：
  - **決策摘要** — 記錄 Phase 1 確認的方向性決定（命名、參數、位置、架構選擇等）
  - **實作步驟** — 具體的檔案異動清單（新增/修改哪些檔案、每個檔案改什麼）
- 產出為獨立 md 文件，不直接進入實作

## Phase 3：Review 規劃

分兩階段檢查：

### 3a：Review 決策摘要
- 確認方向對不對
- 方向不對，產出的內容極有可能無法使用或錯誤，這是成本最高的返工

### 3b：Review 實作步驟
- 確認有沒有遺漏、步驟是否合理

### 調整方式
- **主要方式**：使用者告訴 Agent 要改什麼，Agent 更新 plan md
- **備選方式**：使用者自己改 md，再叫 Agent 讀取

### 結束條件
- 使用者確認 plan OK，明確指示進入實作

## Phase 4：實作

- 開始前先讀取最新檔案狀態（確保納入使用者在前一階段的任何修改）
- 依照 plan md 逐步實作

## Phase 5：Review 實作

- 使用者檢查實作結果，可能進行微調（CSS、排列順序、命名等細節層面）
- 可能跟 Agent 討論修正
- 完成後作為 checkpoint：如果接續下一輪任務，下一輪的 Phase 4 需要先讀取這輪的修改

## Phase 間的邊界

- 每個 Phase 的切換由使用者明確指令觸發
- Agent 不應自動跳到下一個 Phase
