# Dev Workflow 優化計畫

## 優化方式

透過實際執行 Dev Workflow 來驗證流程設計，以迭代方式持續改進。

### 執行循環

1. **Run Workflow** — 使用一個具體的開發任務，完整走過 Dev Workflow 的各步驟
2. **記錄問題** — 過程中遇到的問題、模糊地帶、不順暢之處，記錄於改善建議中
3. **產出改善建議** — 針對問題提出具體的修正方向與建議
4. **更新 Workflow** — 確認改善方向後，修正 workflow 相關文件
5. **下一輪驗證** — 再次執行，驗證修正效果

### 測試素材

不固定測試素材。每輪執行時自由選擇開發任務作為測試案例，並在該輪的改善建議文件中記錄素材摘要。

記錄內容包含：
- 素材來源（例如：Jira ticket、需求文件、自訂情境）
- 任務類型（例如：新功能、bug fix、重構、功能擴充）
- 複雜度概述
- 是否有既有 codebase

### 產出文件

每輪執行後產出：

| 文件 | 位置 | 用途 |
|------|------|------|
| 改善建議 | `.improve/改善建議-round-{N}.md` | 該輪的素材摘要、執行中發現的問題、交流紀錄、改善方向 |
| Session Recap | `.recap/RECAP.md` | 更新當前進度，供下次 session 延續 |
