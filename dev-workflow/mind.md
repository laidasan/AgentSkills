# Dev Workflow

開發流程


## 步驟一: 確認有哪些 tasks 
確認要執行的 tasks , 預設一定會搜尋該專案根目錄的 .tasks/ 目錄.

---

## 步驟二: (Human-in-loop) 確認要執行的 tasks ( optional )
確認要執行的 tasks.  
optional 原因： 使用者可選擇跳過的步驟, 跳過時，資料來源默認使用 .tasks/ 目錄下的所有 tasks , 若 tasks 沒有標記完成，則默認需要執行.  

---

## 步驟三：確認現有 Context 的開發原則/規範

從現有的 Context 中，尋找開發原則與規範

---

## 步驟四：(Human-in-loop) 確認開發原則/規範 ( optional )

確認開發原則與規範.  
optional 原因： 使用者可選擇跳過的步驟, 可能既有 Context 就已存在，或是使用者啟用流程時有提供.
或是沒有開發原則/規範.

---

## 步驟五：依照開發規範執行 task

依照開發規範執行 task.  

---
